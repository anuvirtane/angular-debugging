Data Binding Angularissa

Data Binding vaikuttaa DOMiin

Aloitetaan karusti yksinkertaistaen ajattelemalla, että data binding on tapa muuttaa vaikkapa jonkin elementin sisältöä DOMissa (Document Object Model).

Ok, aloitetaan siitä Liian yksinkertaistettua

Data Bindingin voi jakaa karkeasti kolmeen alalajiin sen perusteella, mihin suuntaan tietoa ohjataan:

  1. Source-to-view (yhdensuuntainen, ohjelmalta käyttäjälle)
  2. View-to-source (yhdensuuntainen, käyttäjältä ohjelmaan)
  3. View-to-source-to-view (kahdensuuntainen, käyttäjältä ohjelmaan ja takaisin)


Tässä oppimistehtävässä olen poiminut tekemistäni Angular-harjoituksista yhden esimerkin jokaisesta eri Data Bindingin alalajista. Alla näet kolme screenshotia koodista sekä erittelyn siitä, mitä Data Bindingilla tehdään näissä tapauksissa.

String interpolation

Interpolation

Tuplakiekurasulkujen väliin kirjoitettuna {{ jotain, jonka sisältö on määritelty muualla }} näytetään tässä tapauksessa html-komponentin elementin sisältönä (innerHTML).

Alalaji: source-to-view
Event

Event

(Sulkuihin kirjoitettu tapahtuma), tässä tapauksessa käyttäjän klikkaus, kutsuu funktiota, joka on määritelty muualla.

Alalaji: view-to-source
Bananas in box

Two-way

[(Banaanilaatikkoon asetettu Angular directive)] lähettää tässä tapauksessa käyttäjältä saadun merkkijonon ohjelmalle, joka laittaa sen sinne, missä {{ tuplakiekurasulut ovat }}

Alalaji: view-to-source-to-view
